Throughput

The rate at which the sending process can deliver bits to the receiving orocess.

Bandwidth-sensitive applications

Applications that have throughput requirements.

Elastic applications

Applications can use as little, or as much throughput as it happens to be available.

Timing

A transport-layer protocol can also provide timing guarantees.

Security

Encryption, data integrity, end-point authentication



Transport Services Provided by the Internet

TCP Services

  • Connection-oriented Service -- Client and server exchange transport-layer control information with each other BEFORE the application-level message begin to flow
  • Reliable Data Transfer Service -- Communicating process can rely on TCP to deliver all data sent without error and in the proper order

UDP Services

UDP is lightweight and provides minimal services. UDP prodvides no guarantee that the message will ever reach the receiving process. Also, messages tend to arrive out of order.

Web and HTTP

World Wide Web

HyperText Transfer Protocol - HTTP

  • A client program and a Server program
  • HTTP define the structure of the messages and how the client and server exchange the messages
  • HTTP is a stateless protocol - The server maintains no information about the clients
  • Round-Trip Time -- which is the time it takes for a small packet to travel from client to server and then back to the client
