Global Routing Alogrithm

  • Computes the least-cost path between a source and destination.
  • Uses global, complete knowledge about the entire network
  • The algorithm takes the connectivity between all nodes and all link cost as inputs
  • The algorithm reuires all this information before performing the calculation
  • Link-State (LS) Algorithm

Decentralized Routing Algorithm

  • Least-cost path is carried out in an iterative, distributed manner
  • Nodes only have knowledge of their own directly attached links
  • No node has complete information about the cost of all network links
  • Using an iterative process of calculation and exchange of information with its neighboring nodes
  • A node gradually calculates the least-cost path to a destination or set of destinations
  • Distance-Vector (DV) Algorithm


Link-State Algorithm

  • In practive, this is accomplished by having each node broadcast link-state packets to all other nodes in the network
  • Dijkstra's Algorithm
  • Upon termination, the algorithm will have calculated the shortest paths from the source node to every other node in the network

Distance-Vector Algorithm

  • Iterative, asynchronous, and distributed. Each node recieves some infomraiton from one or more of its directly attached neighbor
  • Bellman-Ford Algorithm
  • The algorithm initializes the distance to the source to 0 and all other nodes to infinity. Then for all edges, if the distance to the destination can be shortened by taking the edge, the distance is updated to the new lower value.

A Comparison of LS and DV Routing Algorithms

  • Message Complexity
  • Speed of Convergence
  • Robustness

Neither algorithm is an obvious winnder over the other. Both algorithms are used in the internet.
