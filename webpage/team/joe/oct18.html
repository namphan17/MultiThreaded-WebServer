Dijkstra's Algorithm

Djikstra's algorithm (named after its discover, E.W. Dijkstra) solves the problem of finding the shortest path from a point in a graph (the source) to a destination. It turns out that one can find the shortest paths from a given source to all points in a graph in the same time, hence this problem is sometimes called the single-source shortest paths problem.

Least Effort / Lowest Cost


Steps to create Dijkstra's Algorithm

  1. Initialization of all nodes with distance "infinite". Initalization of the starting node with 0.
  2. Marking of the distance of the starting node as permanent.
  3. Setting of starting node as active.
  4. Calculation of the distanced of all neighboring nodes by summing up its distance with the weights of the edges
  5. If a calculated distance of a node is smaller than the current one, update the distance and set the current node as antecessor. **Dijkstra's Central Idea**
  6. Setting of the node with the minimal distance as active. Mark its distance as permanent
  7. Repeat steps 4 - 7 until there aren't any nodes left with a permanenet distance, with neighbors that have temporary distances
