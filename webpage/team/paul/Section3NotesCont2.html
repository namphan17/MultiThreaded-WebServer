Notes on Section 3

  1. How is congestion control done?

    • Congestion control within a network is done one of two ways (usually), the most common being an end-to-end system where the router simply sends a bit to the host indicating that there is congestion within the network. Otherwise it's handled by the network passively, by simply injecting packets with indicators that alert the host to network congestion as the are passed from the receiver.

  2. How does TCP manage network congestion?

    • TCP manages network connection by altering how many packets it sends based on the estimated delay for the network congestion. Basically, TCP will alter the increment its MSS by 1 or more based on the calculations done CWND (Congestion Window). This approach towards network congestion is called Slow Start.

    • If the TCP notices a lack of delay, it will increase it's MSS at a continuous rate which is defined by the sender. Otherwise, it's MSS will fall back to the default, and continously rise again based on the delay (so as to detect when congestion has ended).

    • If TCP is sent into congestion avoidance, it will increment the MSS at a very slow, fixed rate. For instance, by finding that the delay has increased, it increases the MSS size by 1. TCP can be thrown into congestion avoidance if it receives multiple ACKs

    • Because TCP monitors is congestion window size automatically based on acknowledgements from routers or hosts, it's known as a self-clocking protocol. Self-Clocking meaning that TCP's values needn't be altered manually by the user, rather the calculations for delay and the adjustments for congestion that follow are all done automatically by the protocol itself.

  3. UDP congestion control?
    • Unlike TCP, UDP doesn't implement internal congestion control within the protocol. Rather, if congestion control were implemented, it would have to be done via the hosts or the network purposely and manually. Such as by doing end-to-end congestion avoidance, or network controlled congestion avoidance as mentioned above.
