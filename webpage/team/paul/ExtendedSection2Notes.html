Notes on Emails and File Sharing Protocols

SMTP

  • Client Connects to SMTP Server, creates message, and queues message to be delivered to another clients SMTP Server. The other client will then connect to their email server to read the emails, and respond.

  • If Client Server is down, or message cannot be transmitted, then the email will be sent as soon as the SMTP Server comes back online (by retrying the connection every 30 min or so).

    • Example of SMTP Connection:


    telnet gud.mystmpserver.domain 25 EHLO ("Hail" the server, this isn't always needed).

    HELO gud.mysmtpserver.domain

    "Response: Hello from gud.mysmtpserver.domain, nice to to meet you [IP address]"

    MAIL TO: root@mysmtpserver.domain

    "Response: 250 root@mysmtpserver.domain... Sender ok"

    RCPT TO: somebody@something.domain

    "Response: 250 somebody@somethingr.domain... Recipient ok"

    DATA

    (message here)

    .

    "Response: 250 Message accepted"

    QUIT

    "Response: 221 gud.mysmtpserver.domain closing connection"



    POP3 and IMAP


  • Similar to SMTP in terms of how emails are handled: POP3 is very simple in design, yet is limited while IMAP is more complex.

  • POP3 has a download-and-delete method, where a user who reads an email from one machine will be unable to read it from a different one due to the message being removed.

  • Remote folders cannot be created to store messages in with POP3.

  • IMAP provides more functionality than POP3, which can store messages in remote folders, manage an inbox more efficiently, etc.



    • FTP


  • Used to share files P2P, and controls data out-of-band as opposed to HTTP's in-band data control. This means that the FTP protocol uses separate control connections to send files as opposed to sending header information and other data within the same connection sending the file. HTTP, on the other hand, sends header information as well as other identifiers along with the file in the response, which is a in-band connection.

  • When connecting to an FTP Server, the client will login using the commands using the following commands:

    USER username

    PASS password

    The the user can display the files to be grabbed by using the command LIST (newer ftp servers also accept the unix command "ls"), and download, upload, or delete files by using GET, DELETE, or PUT.

  • If the client would like to close the connection, they simply type "BYE".
