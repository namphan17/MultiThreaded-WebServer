Shared Notes: How bad guys compromise networks and internet applications (From Section 1.5)

What types of attacks can malicious attacker use to compromise a network? Here are some of the most common ways:

  1. DDoS/DoS (Distributed Denial of Service/Denial of Service)
    1. Bandwidth Flooding: Sending many packets to a target host in order to prevent any other users from using the host system.
      • Example (very old): Pinging a server with the maximum amount of bytes from X amount of computers (or a single computer) to flood the host server with data.
    2. Connection Flooding: Creating a large amount of connections (usually via TCP)to a target host in an attempt to eat up all available connections so that the host system cannot allow any more connections from users.
      • Example: Using a botnet of slave computers to request (and possibly connect) to a website X amount of times each, eating up all of the available connections.
    3. Vulnerability Attack: Targeting a known vulnerability on a host system by sending a specialized set of packets, or information within the packets, that crashes the host's system.
      • Example: A host application can only take X amount of characters as input, anything more will cause the program to crash
  2. Malware (Worms, Virii (Viruses), and Trojans oh my!)
    1. Worms: Malware that targets host systems that are exploitable in certain ways without any user interaction, and replicates by targeting similar systems once inside of a host system.
    2. Virii (Viruses): Malware that gets onto a system with some user interaction, usually via an email, phishing (social engineering), or by visiting a malicious site. Once inside a host system, it continues to replicate by phishing from the host (usually via email) or by making the host a slave.
    3. Trojan Horse Programs (not covered): Malware that is used to give backdoors into host systems so that the attacker, or even multiple attackers, can use the computer for credential theft, phishing, spam, and more. The way this type of malware works is similar to the original story of the Wooden Trojan Horse, as trojans disguise themselves as legitimate programs, or are implanted in seemingly harmless programs.
  3. Packet Sniffing
    • Usually Done via a MITM (Man in the Middle Attack via arpspoofing, DNS hijacking, etc.), this involves the attacker running a packet sniffer (looking at you, wireshark!) in order to gain sensitive information about the victim(s). This could include passwords, file downloads, VoIP (Voice of IP calls), and more. Password stealing requires the attacker to be running an SSL stripper, which takes away the secure certificate of the site and enforces http over https (passwords can be seen in cleartext this way).
  4. IP Spoofing
    • This is done by manipulating the packets sent so that the source IP address (where the information from the packets is coming from, and where it's going to be sent to be received) is modified so that a different IP is used in place of the attackers IP. This could be done for a multitude of reasons, however the biggest ones can be to perform incomplete Syn-Ack DoS/DDoS attacks and to mask the attackers IP address.

There are other ways attackers can compromise hosts (via various forms of exploitation, phishing, ransom-ware, spam, etc), but these will be covered later!
