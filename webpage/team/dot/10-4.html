Notes for October 4th

  • Layering in general
    • Process divided into steps which are all performed independently of each other
    • Each step must be completed before the next step may begin
    • Allows for one step's implementation (though not function) to change without affecting the other steps/layers
    • Layers may duplicate lower-layer functionality
    • Layer may need information only available in another layer
  • Service Model
    • Defined by service offered by each layer
    • Each layer performs certain actions within the later and/or uses services of the layer below
    • Layer can be implemented in software, hardware, or a combination
    • All layers together are called the Protocol Stack
  • Five-layer Internet Protocol Stack
    1. Application
      • Ex: HTTP, SMTP, FTP
      • Nearly always implemented in software
      • Example function: DNS (Translates human-friendly names into computer-friendly addresses)
      • Distributed over multiple end systems
      • Application in one end system uses protocol to exchange packets (messages) with other end systems
    2. Transport
      • Nearly always implemented in software
      • Transports application-layer messages between endpoints
      • TCP
        • Connection-oriented
        • Guaranteed delivery of messages
        • Flow control
        • Breaks long messages into segments
        • Provides congestion control
      • UDP
        • Connectionless
        • No frills: none of TCP's listed benefits
      • Packet referred to as a segment
    3. Network
      • Often mixed hardware and software
      • Moves packets called datagrams between hosts
      • Passed a segment and destination by transport layer
      • Delivers segment to designated destination
      • IP Protocol
        • Defines fields in datagram
        • Defines how end systems/routers act on fields
        • Only one IP Protocol exists
      • Routing protocols which determine routes datagrams take
      • Administrator runs one of many routing protocols
      • Often referred to as IP layer
    4. Link
      • Typically implemented in network interface card
      • Routes datagrams through routers
      • Network layer passes datagram to link layer
      • Link layer passes datagram to next node and back to network layer
      • Protocol examples: Ethernet, WiFi, DOCSIS (cable)
      • Each datagram may use multiple link protocols during its journey
      • Link layer packets are frames
    5. Physical
      • Typically implemented in network interface card
      • Moves individual bits from the packet
      • Protocol varies depending on medium
      • Ex: twisted-pair copper wire, fiber optics
  • Seven-layer International Organization for Standardization Open Systems Interconnection Reference Model
    1. Application
    2. Presentation
      • Provides services allowing applications to interpret the data
      • Data compression, encryption, and description
    3. Session
      • Delimits and synchronizes data exchange
      • Provides means to build checkpointing and recovery
    4. Transport
    5. Network
    6. Link
    7. Physical
  • Modern Internet uses 5-layer
  • Functions unique to the 7-layer model are the responsibility of the application developer
