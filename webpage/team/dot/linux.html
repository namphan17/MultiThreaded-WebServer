Linux Terminal Basics

Open the terminal

  • Click the "Terminal" icon (grey box filled with black with an arrow and underscore inside) in the sidebar
  • Search the computer for "Terminal"
  • Keyboard shortcut: Ctrl+Alt+T

Basic Information

  • Press the up arrow on the keyboard to see the previously entered command
  • If a directory or file name contains a space, it must be either escaped out (file\ name) or contained in quotes ('file name' or "file name")
  • File names include the extension: 'file.txt' and 'file' are treated by Linux as two separate objects
  • If you are ever asked for a password, it will not appear in the terminal as it is typed
  • Press Tab to autocomplete a file or directory name. If more than one name is possible from the characters entered, it will complete as much as possible without knowing which you wish to select. (Ex: 'tyler a' vs 'tyler b', pressing Tab after entering 't' will give you 'tyler ')
  • Copy: Ctrl-Shift-C
  • Paste: Ctrl-Shift-V
  • ? can be used to replace any one character
  • * can be used to replace any number of characters, including zero
  • man [command] will provide a manual for that command and its uses, though not all commands have a manual
  • All commands are case-sensitive
  • File names are displayed in white
  • Directory names are displayed in blue

Common Commands

  • clear
    • Clears the console
    • Previous commands are still stored, simply no longer visible
    • Keyboard shortcut Ctrl+L
  • pwd
    • Give the full path of the current directory
  • ls
    • List all files and directories within the current directory
    • ls -a to list hidden files as well
  • locate
    • Find all matching files and return their paths
    • locate *.png returns all .png image files
    • locate test.* returns all files, whatever the file type, which are named 'test'
  • cd
    • Change Directory
    • cd .. to go up a directory
    • cd [sub-directory]
    • cd [sub-drectory]/[sub-sub-directory]
    • cd ~/ to go to the root directory
    • These commands can be arranged and stacked as much as needed to define the complete directory path
  • mv
    • Move a file to a new location
    • Analogous to cut, not copy
    • File location defined in the same manner as in cd
    • mv [file path] [path to new directory]
    • mv [filename] [new filename] to rename a file
    • If the file is in the current directory, only the name is needed
    • If the file is in a different directory, its filepath must be specified
    • mv [path]/[file] . to move a file into the current directory
    • Can also be used to move entire directories at once
  • cp
    • Copies a file
    • cp [filename] [new filename] within the same directory
    • cp [filename] [path]/[new filename] into a new directory
    • If the file is moved to a new directory, the new filename may be the same as the original
  • rm
    • Delete a file
    • rm [file] will delete the specified file
    • File path specified in the same manner as mv
    • rm -r [directory] to delete an entire directory and all of its contents
    • rm *.png to delete all .png files in the current directory
  • mkdir
    • Create a new directory
    • mkdir [name] creates a sub-directory within the current directory
    • Can be used in conjunction with directory paths
    • Ex: mkdir ~/[name] creates a sub-directory within the root directory
  • touch
    • Create a new file of the given type
    • touch [file].[extension]
    • If the file already exists, it will not be changed, but the 'last modified' timestamp will be updated

Advanced Commands

  • echo
    • Insert text into the console or a file
    • echo "[text]" prints text to the console
    • echo "[text]" > [file] inserts text into a new file
    • If [file] is the name of a pre-existing file, its contents will be completely overwritten
    • echo "[text]" >> [file] appends text to the end of a file
    • > and >> can be used with other commands (ls, locate, etc.) to save terminal output to a file
  • more and less
    • Shows results of a command which has a long result in a more readable way
    • [command] | more shows results one page at a time
    • Cycle through pages by pressing Space
    • [command] | less shows the first page of results and allows scrolling
    • Scroll through pages with the up and down arrow keys
    • Exit both less and more by pressing q
  • wc
    • Gives the wordcount of a file
    • wc [file]
  • PS1
    • Change the command prompt
    • PS1='==>' sets the command prompt to ==>
    • Can be done with single or double quotes
    • Once the command prompt is set, the current folder hierarchy will no longer be shown as part of the prompt
    • To reset the prompt to the default, enter PS1='\e[1;32m[\u@\h \W]\$ \e[m '
      • \e[1;32m defines the color of the prompt
      • \u@h inserts username and host name
      • \W puts the directory path at the end of the prompt
      • \$ displays the user ID, or a $ if the effective UID is 0
      • \e[m marks the end of the color range
