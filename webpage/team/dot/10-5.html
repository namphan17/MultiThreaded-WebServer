Notes for October 5th

  • Application Architecture
    • Designed by application developer
    • Dictates how application is structured over various end systems
    • Two main models: client-server and peer-to-peer
  • Client-server
    • Always-on host (server) services requests from many hosts (clients)
    • Ex: Web, FTP, Telnet, e-mail
    • Server receives request and responds by sending the requested object to the client
    • Clients do not directly communicate with each other
    • Server has a fixed, well-known unique IP address
    • Client sends packets to server's IP
    • When a single-server host is insufficient, many servers can be connected in a data center
  • Peer-to-peer
    • Often abbreviated P2P
    • No single dedicated server
    • Distributed between pairs of intermittently connected hosts, peers
    • Peers are owned by users, not the service provider
    • Ex: BitTorrent, Xunlei, Kankan, PPstream
    • Significant problems
      • Not ISP friendly: modern residential ISP configured for 'assymetric' usage, not equal uploading and downloading
      • Difficult to secure: distributed and open by nature
      • Require incentive: must convince users to volunteer resources to the operation of the P2P system
  • Hybrid example: IM applications
    • Server used to track IP addresses of uses
    • Messages sent directly with P2P
