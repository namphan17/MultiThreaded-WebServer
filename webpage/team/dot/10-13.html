Notes for October 13th

  • Some people use Routing and Forwarding interchangeably, but this text makes a distinction between the two
  • Routing: planning the entire route a packet will take.
  • Real-life analogy: planning your route on Google Maps before leaving the house
  • Forwarding: deciding where the packet will go next from the current router
  • Real-life analogy: deciding which exit to take on an interchange to reach your destination
  • Packet switch: general packet-switching device which transfers a packet from input to output link based on the value in a header field
  • Link-layer switches: base forwarding decision on values in fields of the link-layer frame
  • Router: base forwarding decision on value in network-layer field
  • Connection setup: third important function of the network layer
  • In TCP, the connection setup is the 3-way handshake
  • Jitter: variability of end-to-end delay
  • Network service model: defines characteristics of end-to-end packet transport between sender and receiver
  • Specific services may include:
    • Guaranteed delivery: guarantees packet will eventually arrive
    • Guaranteed delivery with bounded delay: guarantees packet delivery within a specified timeframe
  • Possible services offered to a flow of packets:
    • In-order packet delivery: guarantees packets will arrive in the order they were sent
    • Guaranteed minimal bandwidth: as long as the sender transmits at a rate (bits/sec) below the specified rate, no packet is lost and each packet arrives within a specified delay
    • Guaranteed maximum jitter: guarantees that the amount of time between transmission of packets is equal to amount of time between their arrival at the destination (within a specified margin of error)
    • Security services: uses a session key known only by receiver and sender, sender can encrypt the packets so only the receiver may read them
    • Security may also include data integrity and source authentication
  • Best-effort service: timing not guaranteed, order not guaranteed, eventual delivery not guaranteed, but it does its best
  • Two types of ATM network services
    1. Constant Bit Rate
      • First ATM model to be standardized
      • Provides a flow of packets with a virtual pipe whose properties allow the assumption that a dedicated fixed-bandwidth transmission link exists between sender and receiver
      • End-to-end delay, jitter, and fraction of packets lost or delivered late are all guaranteed to be less than specific values established when the connection opens
    2. Available Bit Rate
      • "Slightly-better-than-best-effort" service
      • Packets may be lost, but they cannot be reordered and minimum packet transmission is guaranteed
      • If the network has the resources, senders may transmit above minimum transmission rate
      • Can provide feedback to control how sender adjusts rate between minimum and allowable peak packet transmission rate
  • Virtual-Circuit networks: computer networks which provide only a connection service at the network layer
    • Three parts:
      1. a path between source and destination hosts
      2. a VC number for each link along the path
      3. entries in the forwarding table in each router along the path
    • Replaces part of the packet header with the next VC number at each router as part of the forwarding process
    • Maintains connection state information
    • Connection state information: each time a new connection is added or released across a router, the forwarding table must be updated
    • Three phases in a connection:
      1. VC setup: transport contacts network layer, specifies receiver's address, network layer determines path packet will travel and makes entries in forwarding tables
      2. Data transfer: packets begin to flow
      3. VC teardown: one side tells the other it wishes to stop, network layer passes it on up, remove information from forwarding tables
  • Datagram networks: computer networks which provide only a connectionless service at the network layer
    • Packet is stamped with final destination and thrown into the network/wind
    • Each router compares the destination to its forwarding table to decide where it should go next
    • Forwarding table generally matches prefixes, putting priority on longer prefixes because they're more specific
  • Forwarding function: actual transfer of packets from incoming to outgoing link; sometimes referred to as the router forwarding plane
  • Components of a router:
    • Input port: connects to the link/physical layer, where the forwarding table is consulted to see where a packet is headed; control packets go straight to the routing processor
    • Switching fabric: internal network connecting input and output ports within the router
    • Output port
      • : stores packets from switching fabric and sends them to the link/physical layers
    • Routing processor: executes routing protocols, maintains routing tables and link state information, computes forwarding table
  • Packet scheduler: chooses a packet from the output queue to be transmitted (FIFO, weighted fair queuing)
  • Drop-tail: policy of dropping arriving packets when the buffer is full
