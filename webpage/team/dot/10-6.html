SMTP

  • Defined in RFC 5321
  • Transfers messages between sender and recipient mail servers
  • Older than HTTP
  • Age lends itself to archaic restrictions which have yet to be phased out
    • Body and headers of all messages restricted to 7-bit ASCII
    • Fun fact: part of my father's job used to be making sure email signatures were less than 3 lines long and therefore able to be handled by the server
  • Order of operations
    1. User A invokes user agent for email, provides email address of user B, writes a message, and instructs user agent to send it
    2. User A's user agent sends the message to the mail server, where it is placed in a message queue
    3. Client side SMTP on User A's mail server sees the message in the queue and opens a TCP connection to an SMTP server on User B's mail server
    4. SMTP client and server initialize a connection and the client sends the message
    5. Server SMTP receives the message and places it in User B's mailbox
    6. User B invokes user agent to read the message from User A
  • SMTP does not use intermediate mail servers
  • If the destination server is down, the sending server holds onto the message until it can be sent
  • SMTP uses port 25
