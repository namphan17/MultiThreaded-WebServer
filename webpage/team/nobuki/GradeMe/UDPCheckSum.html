UDP Checksum

  • The purpose of the UDP checksum is for error detection. It makes sure that there is not an altered value within the UDP segment.
  • It does this by sending the 1's compliment of the sum of all the 16-bit words in the segment
    • If there is an overflow, it is wrapped around.
  • The UDP provides a checksum because there is no garuntee that there will be error checking in between the source and destination.
  • Also bit errors can be introduced anywhere including when a segment is stored in a router.
  • The checksum is an error detection that works on an end-end principle.
  • End-end principle is what it sounds like. It is a check that works on both ends.
  • It is redundant to check in between the end-end basis.
  • UDP checksum only checks for errors
    • It does not recover errors.
    • Since it works with the sum, there is no way to garuntee what was sent. Also, the checksum itself could have an error.
    • When an error is detected, the dammaged segments could be discarded or it forwards the segment with a warning.
