Broadcast Routing Algorithm

  • N-way-unicast approach: Given N destination nodes, the source node makes N copies of the packet, addresses each copy to a different destination and then transmits the N copies to the N destinations using unicast routing.
    • This is simple because no new network-layer routing protocol, packet-duplication, or forwarding functionality is needed.
  • It is very inefficient: it would be easier to just send one copy of the same packet instead of N copies.
  • There is unnecessary overhead because of the extra copies
  • There are three ways of doing this.
  • Uncontrolled Flooding
    • Flooding is the easiest way of broadcasting. One node sends to all its neighbors, then those neighbors pass it on to their own neighbors
    • Broadcast Storm: On a large scale, there would be so many broadcasts, that the packets would become useless.
  • Controlled Flooding
    • This chooses when to flood a packet more carefully.
    • Sequence-number-controlled flooding: a source node puts its address as well as a broadcast sequence number into a broadcast packet then sends the packet to all of its neighbors.
    • Reverse Path Flooding a router transmits the packet on all of its outging links only if the packet arrived on the link that is on its own shortest unicast path back to the source.
  • Spanning Tree Broadcast.
    • Spanning Tree: a tree that contains each and every node in a graph.
    • Minimum Spanning Tree: each link has an associated cost and the cost of the tree is the sum of link costs. That cost is the minimum of all the graphs trees.
    • The Spanning Tree Broadcast will use the minimum spanning tree to find the best way of sending data.
