Reading (4.4)

  • Datagram format IPv4
    • Version number: 4 bits. specifies the IP protocol version, tells the router how to interpret the rest of the datagram.
    • Header length: 4 bits. Determine where in the IP datagram the data begins. Typical header length is 20-bytes.
    • Type of service: Allow different types of IP datagrams. Mainly to destinguish between real-time datagrams and non-real-time datagrams.
    • Datagram length: Total length of the IP datagram (header and the data) in bytes. 16 bis long, maximum size is 65,535 bytes.
    • Identifier, flags, fagmentation offset: IP fragmentation. IPv6 does not allow for fragmentation.
    • Time-to-live: Keeps datagrams from circulating forever. Decremented each time it passes though a router, if it reaches 0 the datagram is dropped.
    • Protocol: Indicates the specific transport-layer protocol that the datagram should be passed to. Helps bring together the network and transport layers.
    • Header checksum: Helps detect errors in the datagram. Treats each 2 bytes in the header as a number and sums the numbers with 1s complements. If there is a detected error the datagram is discarded. The checksum is recomputerd and stored again at each router.
    • Source and destination IP address: The IP address of the source of the datagram and its intended destination.
    • Options: Allow an IP heade to be extended. Dropped in IPv6.
    • Data: The main contents of the datagram, the data that is being sent.
  • Maximum Transmission Unit (MTU): Maximum amount of data a link-layer fram can carry.
  • Fragment: To avoid having datagrams exceed the MTU they are ofen broken up into smaller datagrams called 'fragments'. Fragments have incrmented id numbers to show the order. Last fragment has flag set to 0, while rest set to 1.
  • Interface: Boundry between a host and the physical link to the network. Each has their own IP address.
  • Dotted-decimal notation: How IP addresss are written. Each IP address is 32 bits (4 bytes). Each byte of the address is written in decimal form seperated by a dot (.).
  • Subnet: Router connected multipul devices to the interent under IP address
  • Subnet mask: IP address assigned to a subnet
  • Classless interdomain routing (CIDR): Internet addressing strategy. Generalizes subnet addressing. IP address in form of a.b.c.d/x, with x being numbe of bits in first part of address.
  • Prefix: x most significant bits of an address.
  • Classful addressing: Before CIDR IP addresses were 8, 16, or 24 bits. According to their class.
  • Dynamic Host Configuation Protocol (DHCP): Allows a host to obtain an IP addres automatically. can be configured to regularly give same IP address, or a temporary one that changes each time.
  • DHCP server discover: Sends a discover message to port 67 in a UDP packet. It is a datagram with destination IP of 255.255.255.255 and a souce IP of 0.0.0.0.
  • DHCP server offers: A DHCP server that receives a discover message (ACK) replies with a offer message. It broadcasts with IP address 255.255.255.255. Contains the transaction ID of the received discover message. Also contains the IP address lease time.
  • IP address Lease time: How long an IP address is valid.
  • Network address translation (NAT): An interface that acts as a filter between a home network LAN, and the outer WAN. To the WAN it has a single IP address, while it actually connects multipule hosts on the LAN side. Uses a NAT tanslation table to determine where to address outside WAN requests.
  • NAT translation table: Allows the NAT router to direct incoming WAN requests/messages. Includes port numbers as well as IP addresses.
  • Complaints about NAT:
    • Port numbers are meant to be used for addressing processes, not for addressing hosts.
    • Routers are supposed to process packets only up to layer 3.
    • NAT protocol violates the end-to-end argument.
    • We should use IPv6 to solve the shortage of IP addresses
