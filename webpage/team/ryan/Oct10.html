Lecture

  • Multipul ways to itentify something. In certian situations one way might be more better than another to use to identify.
  • hostname:Mnemonic way to identify an address. Useful for people, but not computers. Tells little about computer/server's location. Made of variable alphanumeric characters.
  • IP addres:IP numbers are the social security numbers to websites. Decimal formatted, 4 numbers, 0-255 seperate by a period. (ie. 0.255.16.255). Heirarchical, read left to right.
  • DNS:(Domain Name System)A data base that links hostnames to IP addresses. Distributed database and an API. Berkeley Internet Name Domain (BIND) uses the DNS. Runs on UDP transport protocol on port number 53 (love bug: 53). Adds delay, usually short, sometimes a little longer. Usually delay is shortened by nearby lower level DNS servers cacheing commonly used urls for fast lookup. Used indirectly.
  • cache: Local memory that stores what you looked up last. Used everywhere to speed up preformance.
  • Host aliasing: A DNS service. When there are multipul host names that point to an IP address it makes sure that they both direct to the correct location or the canonical hostname.
  • Mail server aliasing: A DNS service. Allows the lookup of the canonical hostname of the email address.
  • Load distribution:A DNS service. For large websites with multipul IP addresses it roates which address is used to avoid overloading one server. This balances the amount of users requesting each server.
  • DNS Problems, for simple centralized design:
    • Single point failure: If the DNS server crashes so does the internet (assuming one server).
    • Traffic volume: One server would have to handle all traffic, would become incredibly slow.
    • Distant centralized database: Delays created by DNS server being physically far away.
    • Maitenance: Would have to contain addresses and hostnames for the whole internet, would be huge and require frequent updates.
  • Alternate
    • Root DNS servers: Main root servers that contain whole or most of DNS. Multipul with lower level servers below them. There are 13 root servers atm.
    • Top-level domain (TLD) servers: Responsible for top-level domains, which as .com, .org., .net, .edu, and .gov and country level domains.
    • Authoritative DNS servers: Organizations on the internet had an authoritative DNS server that houses the records of their host names and IP addresses. Such as universities and large companies.
  • DNS message
    • Identification (2 bytes)
    • Flags (2 bytes)
    • # of Questions (2 bytes)
    • # of answer RRs (2 bytes)
    • # of authority RRs (2 bytes)
    • # of additional RRs (2bytes)
    • Questions
    • Answers
    • Authority
    • additional info
  • DNS vulnerabilities
    • DDoS attacks: either sending packets to the root servers in large amounts to slow them down and stop replies. Or send many DNS queires to top-level-domain servers to slow them down.
    • Man in the middle: Intercepts queries from hosts and returns bogus replies. This redirects people to false websites.
    • DNS poisioning: Sends a bogus reply to the DNS server so it caches the false records.
    • Use DNS to create a DDoS against a server, by sending queires to many authoritative DNS servers, each having the spoofed source address of the target host. If packets are larger in bytes than query then they can potentially overwhelm a target.
  • Wireshark
    • Packet sniffing softwear
    • Displays info such as source address, destination address, frames, contents, protocol, etc.
    • Allows you to see traffic between computer and other hosts on the internet
    • Allows you to filter packets by protocol, ip addresses, etc. In real time.
