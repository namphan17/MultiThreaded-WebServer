Lab 1: Intro

General overview of how to open and use wireshark. Wireshark is a packet sniffer application, it captures packets that the host computer either transmits or receives. It displays them in an easy to view format with a list of packets at the top in the packet-listing window, along with their name, type, source, and destination. Then there is the packet-header details window that contains information on header fields and data. Including size and contents. Finally there is the packet-contents window that displays the raw data of the packet in both hexadecimal and ASCII code. Wireshark can also be filtered using the bar at the top. Entries into the filter bar should be lowercase.

Notes:
Your host computer sends a GET message to the destination address to request the information. The destination host should reply with the requested information.
HTTP messages are often carried inside multiple TCP messages.
Browsers often cache requested objects, so the catche must often be cleared to resend/receive packets.
Long/large documents are submitted over several packets that are received and reassembled
Usernames and passwords are encoded in Base64 when transmitted so that they are more secure, they are not encrypted.
