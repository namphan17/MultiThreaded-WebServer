    • Set a single node to be the "center". Serves are root of spanning tree.
    • Tree-join messages are sent from every node in graph (or graph subset). Hop from node to node till it reaches either the center, or a branch that already reaches the center.
    • Nodes not part of the spanning tree, act as hubs for either further nodes trying to join, or messages sent though the spanning three.

  • Link-state (LS) routing algorithm
  • each node floods using link state packets to find all of its neighbors
  • each node receives every other nodes' LSP, combines them into a full topology of the network
  • Each node uses dijkstra's algorithm
  • Makes a forwarding table for each node
  • Flood update sent out and recomputed if there are changes
    • Link missing or removed, then link is removed
    • node missing or removed, then links to that node are removed (removing node by association).
    • if new node is added, or old one updated then LSP updates all of the links
