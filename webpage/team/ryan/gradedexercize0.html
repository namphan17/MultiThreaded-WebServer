Biography on Famous Computer Scientist

Robert "Bob" Elliot Kahn

Born December 23rd 1938 in Brooklyn (yes that one). He went to City College of New York and earned a Bachlors of Electrical Engineering in 1960, followed by his M.A. in 1962 and his Ph.D. in 1964.
Shortly after earning his Ph.D. he worked at AT&T Bell Labs and then joined MIT's engineering department as an assistant professor in 1966.
In 1968 the US Department of Defense began a project called ARPANET to create long-distance communication using telephone lines and packet switches. Kahn's proposal for system design won him the contract and he joined the ARPANET team.
Later in 1972 Kahn joined the Information Processing Techniques Office to work on network security, digital speech transmission, and packet radio. He then began a ground-based packet-radio projected called PRNET based off Alohanet. This along with using a satellite to link Arpanet sites created the SATNET project.
In an attempt to connect the networks that he had created Kahn went to Vint Cerf with an idea of a network to connect the networks. They then wrote a paper called A Protocol For Packet Network Intercommunication which would lay the foundation for what would become the internet.
The internet that they created was made off two key ideas:

First a host protocol called the Transmission Control Protocol(TCP), this provided ordered, reliable, flow-controlled transmission of packets over linked networks.

Second was gates and routers that would direct the traffic of packets, handeling the routing.

In 1977 they finalized the implementation of TCP, and the first incarnation of the internet. And in 1983 they guided the changeover of ARPANET from NCP protocols to TCP.
Now Kahn is the chairman, CEO, and president of Corporation for National Research Initiatives (CNRI), which he founded in 1986. Vint Cerf joined Kahn as Vice President, and they later also founded the Internet Society (ISOC), which helps set international technical standards for the Internet.

Tutorial

Github File Transfer

Using Git to upload to Github (And SSH generation):

First you would want to make an ssh key so you can upload easier and faster to your github. To do this you will need to open up the terminal window. In the terminal type:

ssh-keygen -t rsa -b 4096 -C "[your github account email]"
#generates specified SSH key

This will generate a new ssh key. It will offer you an option of where to save it, hitting enter will just save it to the default file path. It will also prompt for a password, hitting enter will leave this blank.
To copy the public key type into the terminal:

cat .ssh/id_rsa.pub
#shows the contents of the rsa public key

Copy the text and then under your github account settings create a new ssh key, name it whatever you want, and paste the text into the body. This will link your ssh key to your github allowing easier transfers when you CLONE a file to the computer, and PUSH it back up.

If you wish to add a file to your github account you must follow a few steps. First you must initialize the git protocol on your files. Then you have to add the files to the git, and commit them. Finally you should PUSH them up to github. This is done in the following manner:

git init
#initializes the git protocol
git add [filenames/pathway]
#adds the files to git
git commit -m "[notes/comments]"
#commits the files, and adds a note/comment on what changes were made
git push
#PUSHes the files from git to github

This will PUSH the files up to your github account, having a linked ssh will make this faster and easier.

If you wish to FORK over a file in github from another account to your own, you simply need to go to www.github.com search either the other user's account or their project, select the desired project, and hit the provided FORK button. This will copy the project onto your account.
From your own account you can CLONE the project from your github account to your local computer. To do this hit the green CLONE button. Select "clone with SSH" and copy the link provided. In the terminal window type:

git clone [copied link]
#CLONEs the github files from the specified link

If you set up a password you will be prompted to enter it. Otherwise the file will CLONE itself to your machine. Once on your local computer you can edit the files either directly though the file explorer, or through the terminal window with the gedit command.
When you are done editing the files you can PUSH them back to your github using the add and commit commands:

git add [filename]
#adds files (with changes) to git
git commit -m "[notes/comments]"
#commits the added files and adds a note/comment
git push
#PUSHes files back up to github

When it is PUSHed up to your github you can then send a PULL request to the original author by clicking the PULL request button. Be sure to add a comment so they know what changes your made.

This concludes the short overview tutorial on how to transfer files to github, FORK them, CLONE them, edit them, PUSH them back up to github and then send a PULL request.
