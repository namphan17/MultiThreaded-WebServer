Client-Server Architecture

A server is a computer that serves items to other computers. The server is called the host, and the other computers requesting items from the server are called clients. Applications with a client-server architecture include:

  • Web
  • FTP
  • Telnet
  • Email

A data center is a collection of servers. Large internet service providers like google have many datacenters numbering in the hundreds of thousands of servers.

P2P Architecture

In P2P there is no need for servers. applications exploit communication between two connected hosts called peers. Becuase these hosts communicate directly, it is called a peer-to-peer connection. P2P applications include

  • File Sharing (Torrents)
  • Download acceleration
  • Telephony(Skype)
  • IPTV(Internet Protocol Television)

P2P can distribute the workload of large processes over multiple machines. Seti@Home was one of the first public distributed computing projects where users participated in the analysis of space sounds (radio signals). Videogames utilize P2P where the games are hosted by players, as opposed to dedicated servers. Call of Duty Modern Warfare 2 and Black Ops used P2P. If the host left the game the game would wait for a host migration or the players would be disconnected.

Two types of P2P

  • Unstructured
  • Structured

    • An unstructured network has peers randomly connecting to some subset of all peers. This doesn't scale particularly well. The main disadvantage is the need to search the entire network to find something

    A common type of structured P2P is a distributed hash table (DHT), in which a variant of consistent hashing is used to assign ownership of each file to a particular peer
