What is P2P networking?

It is where applications exploit communication between two connected hosts called peers. Becuase these hosts communicate directly, it is called a peer-to-peer connection.

What is a bottleneck?

A node in a network that has a lower throughput than other nodes.

How would one calculate a propagation delay?

It can be computed as the ratio between the link length and the propagation speed over the specific medium.

What is Traceroute?

A utility that records the specific gateway computers at each node through the internet between your computer and the destination.

What is the transmission rate of Ethernet LANs?

10 Mbps

What is a physical form media that Ethernet can run over?

Coaxial Cable - Twisted pair Copper Wire - Fiber Optic

Why are standards important for protocols?

They are important so people can create networking systems and products that interoperate.

What is the difference between a virus and a worm?

Worms are similar to viruses in that they replicate functional copies of themselves and cause the same damage. Viruses requires spreading of an infected host file, and worms are standalone that do not require a host program or human to propogate

Norman Abramson

Abramson created the first wireless local area network. Its birthplace was in Hawaii, hence its name ALOHAnet. It was designed and developed at the University of Hawaii. It was the first network to successfully transmit data using radio signals. Data transmission between islands was a essential and ALOHAnet was the solution. ALOHAnet used high speed data packets, AKA ALOHA channels, to transmit data over radio frequencies. This has been used in every generation of mobile broadband. Abramson's ALOHAnets packet broadcast technology was a revolutionary advance over the switched-circuit data technologies of the time. Ironically enough, Abramson's wirelss technology helped in the design of the Ethernet, which was made by Robert Metcalfe.

Instructions for GitHub

  • Create a new repository: git init
  • Create a working copy of a local repository by:git clone /path/to/repository

    • Your local repository consists of three "trees" maintained by git. the first one is your Working Directory which holds the actual files. the second one is the Index which acts as a staging area and finally the HEAD which points to the last commit you've made.

  • propose changes: git add < filename > or git add *
  • commit these changes: git commit -m "Commit message"
  • pushing changes: git push origin master

    • If you have not cloned an existing repository and want to connect your repository to a remote server, you need to add it with

  • git remote add origin < server >
  • update & merge:git pull

    • in your working directory to fetch and merge remote changes. to merge another branch into your active branch (e.g. master), use

  • git merge < branch >

    • Useful hints

  • built-in git GUI : gitk
  • use colorful git output : git config color.ui true
