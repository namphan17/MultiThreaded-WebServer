Reliable Data Transfer

Implementations of reliable data transfer occur in the transport, link , and application layer. In a reliable channel, no bits are corrupted or lost. They are delivered in the exact order that they are sent.

Reliable Data Transfer Protocol: rdt1.0

  • Finite-State Machine(FSM): an abstract machine that can be in one of a finite number of states
  • rdt1.0: A protocol for a completely relaible channel

There are two different FSMs for rdt1.0. A sender FSM and reciever FSM. Each only have one state. Since There is only one state in each the transitions are right back to the same state. The sending protocol accepts data from the upper layer using rdt_send(data) event and and creates a packet containing the data with make_pkt(data), and sends that packet to a channel. The reciever recieves data through a channel using rdt_rcv(packet), processes the data with extract(packet, data). with a perfectly reliable channel there is no need for the receiver side to provide any feedback to the sender since nothing can go wrong. The receiver recieves just as fast as the faster sends so there is no need for the reciever to tell the sender to slow down.
