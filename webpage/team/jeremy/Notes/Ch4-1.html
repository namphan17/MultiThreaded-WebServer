Where does Queuing Occur?

 Packet queues may form at both the input ports and the output ports, and queuing will happen at each location biased on the amount of traffic, the speed of the switch and the speed of the connection.
  ● Roundabout Analogy

 Queues can only be so big without running out of buffer and packets being lost. Input port queue’s don't form as often as output port queue’s because the computer acting as the receiver is normally faster at processing packets than the network is at transporting packets.



 One of the problems of output port queuing is that the packet scheduler at the output port must choose one packet among those queued to be transmission. This selection might be done using one of several methods:

  • First-come-first-served (FCFS)
  • Weighted fair queuing (WFQ)

 Another problem with output port queuing is that if there is not enough memory to add an incoming packet to the queue than a decision must be made to drop a packet. This can be done by:

  • First-come-first-served (FCFS)
  • Weighted fair queuing (WFQ)

  Ideally this issue gets resolved before it happens using one of the congestion reduction methods we have already talked about.
