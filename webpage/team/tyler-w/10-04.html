The Protocol Stack

Application Layer

The application layer is where network applications and their application-layer protocols reside. The application-layer protocol is distributed over multiple end systems, with the application in one end system using the protocol to exchange packets of information, called messages, with the application in another end system. Some of these protocols include:

  • HTTP, which provides for web-document requests and transfers
  • SMTP, which provides the transfer of e-mail messages
  • FTP, which provides for the transfer of files between two end systems

Transport Layer

The transport layer transports application-layer messages, called segments, between application endpoints. There are two transport protocols:

  • TCP, which provides a connection-oriented service to its applications. It guarantees delivery of application-layer messages to the destination and flow control (sender/reciever speed matching).
  • UDP, which provides a connectionless service to its applications. It provides no reliability, no flow control, and no congestion control.

Network Layer

The network layer is repsonsible for moving network-layer packets, called datagrams, from one host to another. The transport-layer protocol in a source host passes a transport-layer segment and a destination address to the network-layer. The network-layer then provides the service of delivering the segment to the transport-layer in the destination host. The internet's network layer includes the IP protocol, which defines the fields in the datagram as well as how the end systems and routers act on these fields.

Link Layer

The network-layer routes datagrams through a series of routers between the source and destination. To move a packet from one node to the next node in the route, the network-layer relies on the link-layer. At each node, the network-layer passes the datagrams down to the link layer, which delivers the packets, called frames, to the next node along the route. Examples of link-layer protocols include:

  • Ethernet
  • Wi-Fi
  • DOCSIS

Physical Layer

The physical layer moves the individual bits within the frame from one node to the next. The protocols in this layer are link-dependent and further depend on the actual transmission medium of the link. Examples of these mediums are:

  • Twisted-pair copper wire
  • Single-mode fiber optic cable
  • Coaxial cable
