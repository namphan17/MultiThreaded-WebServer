Multiplexing and Demultiplexing

A process can have one or more sockets, which are doors through which data passes from the process to the network. Each transport-layer segment has a set of fields to direct an incoming segment to the appropriate socket. The transport-layer then examines these fields to identify the receiving socket and directs the segment to the socket.

Demultiplexing

  • Delivers data in a transport-layer segment to the correct socket

Multiplexing

  • Gathers data chunks at the source host (from different sockets)
  • Encapsulates each data chunk with header information (used for demultiplexing)
  • Creates segments to pass to the network layer

How does it work?

  • Source port number field: the unique identifier field in a segment
  • Destination port number field: field that indicates what socket to deliver to

Each port number is a 16-bit number, ranging from 0 to 65535. Well known port numbers, ranging from 0-1023, are restricted and reserved for well-known application protocols like HTTP (port no. 80), and FTP (port no. 21). An application must be assigned a port number in order to transfer data.

Connectionless (UDP)

  • Gets assigned a non-well-known port number
  • Chooses an available port number not being used by another application
  • If a port is chosen that is already being used, the return address field sends it back

Connection-Oriented (TCP)

Identified by a four-tuple which uses the four values,

  • Source IP Address
  • Source Port Number
  • Destination IP Address
  • Destination Port Number

    • to direct the segment to the appropriate socket.
