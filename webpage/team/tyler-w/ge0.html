Robert Metcalfe

Robert Metcalfe is an electrical engineer and doctor of computer science who has made huge contributions to the world of data communications. He graduated with a bachelor's degree from MIT in 1969, then went on to get his master's degree and Ph.D. from Harvard University in Applied Mathematics and Computer Science respectively. He is currently a professor at the University of Texas at Austin, in the Cockrell School of Engineering, and was appointed to lead innovation initiatives at the university.

While working at Xerox in the early 1970's, he, along with David Boggs, invented Ethernet - the most widely installed local area network (LAN) technology. Ethernet is a link-layer protocol in the TCP/IP protocol stack, describing how networked devices can format data for transmission to other devices on the same network, and how to put that data out on the network connection. Unlike dial-up networking, Ethernet transmits data at a much higher rate. After the creation of the Ethernet cable, Robert Metcalfe co-founded 3COM, a digital electronics manufacturer best known for its computer network infrastructure products.

In 1980, he received the ACM Grace Hopper Award for his contributions to the development of local networks and the creation of Ethernet. Metcalfe was also awarded the IEEE Medal of Honor in 1996 for "exemplary and sustained leadership in the development, standardization, and commercialization of Ethernet," as well as the 2003 Marconi Award for "inventing the Ethernet and promulgating his law of network utility based on the square of the nodes." In May 2007, he was inducted into the National Inventors Hall of Fame in Akron, Ohio.

Tutorial: The Linux File-System

This tutorial will teach you how to use basic commands to organize your files.

  1. Open up a terminal window. Your current directory should be your personal user folder.

  2. Type ls into the terminal and hit enter. The ls command shows you every file and folder in the current directory.

  3. You should see the folder name Documents in the list. To change to that directory, you would type: cd [directory name], or in this case cd Documents.

  4. You are now in the directory Documents. Open up the file explorer on your operating system and go to the Documents folder. Create a new document by right-clicking the area inside the folder. Rename this document 'testDoc.txt'. Keep this window open.

  5. Return to the terminal and enter ls command again. You should see the testDoc.txt file in the list that appears. Now you will learn how to create a directory.

  6. To make a new directory, type in the command mkdir testFolder.

  7. Type the ls command again, and you should see your created folder in the current directory. If you go back to your OS file explorer window, you will see that the newly created file is in the Documents folder.

  8. Now, back to the terminal window. Now you will learn how to make a copy of a file. To do this, simply type cp testDoc.txt testFolder.

  9. Change your current directory to testFolder by typing cd testFolder, and then type the ls command. You will see that your file was copied to the testFolder folder.

  10. Now you will learn how to delete files. To change your directory back to Documents, use the cd command by typing cd ... The .. refers to the parent folder of the current directory.

  11. You are now back in the Documents directory. Use the ls command and notice that your first copy of testDoc.txt is still there. You can delete this duplicate using the rm command, by typing rm testDoc.txt. Use the ls command again and notice that it is no longer there.

  12. You can use the mv command to move or rename files or folders. Rename the directory testFolder by typing mv testFolder testFolder2. Type ls and see that your directory name has changed. Change your current directory to testFolder2 by typing cd testFolder2. Notice that your testDoc.txt file is still in the directory. You can use the same instructions for files.

  13. You just learned how to rename files and folders, now you will learn how to move them. The only difference here is that the second parameter you enter will be a filepath instead of a directory or file name. In your current directory, type mv textDoc.txt ... This will move your textDoc.txt file out of the testFolder2 directory, and up one level into the Documents directory.

  14. Move up one directory by typing cd .., and see that your file was moved.

  15. This completes the tutorial for basic Linux file system commands.

Questions

What is an Application Programming Interface?

An Application Programming Interface, or API, is a set of routines, protocols, and tools for building software applications.

What does an API do?

An API specifies how software components should interact, and they are used when programming GUI components.

What is instantaneous throughput?

Instantaneous throughput is the rate (in bits/sec) at which the receiving host is receiving the file at any instant of time.

How many layers are in the Protocol Stack, and what is each layer?

There are five layers: Application, Transport, Network, Link, and Physical.

What is the difference between a virus and a worm?

A virus requires some form of user interaction to infect the user's device; a worm does not.

What is IP spoofing, and why is it used?

IP spoofing is when malicious users inject packets into the Internet with a false source address, for the purpose of hiding the identity of the sender or impersonating another computing system.

What is an elastic application?

An elastic application is one that makes use of as much, or as little, throughput as happens to be available. (i.e. E-mail, file transfer, web transfers)

What is the difference between a persistent connection, and a non-persistent connection?

A persistent connection is when all the pairs of requests and responses are sent over the same connection. A non-persistent connection is when they are sent over seperate connections.
