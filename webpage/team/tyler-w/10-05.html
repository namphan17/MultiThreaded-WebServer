Client-Server and Peer-To-Peer Applications

What is a Client-Server Application?

This is a type of application where a host that is always running, called a server, services requests from many other hosts, called clients. These clients do not communicate directly with each other, but with the server itself. The server must be appropriately scaled to the expected workload. An example a of client-server application is the website for a bank, where the client requests data from the bank's servers.

What is a Peer-to-Peer Application?

In this type of application, two or more computers, or peers, pool their resources and communicate in a decentralized network, meaning that they connect to each other rather than a server. A benefit of using peer-to-peer, or P2P, applications is its self-scalability; however there are many set backs:

  • Not ISP Friendly: Most residential ISPs have been dimensioned for "asymmetrical" bandwidth usage.
  • Security Flaws: P2P connections are very vulnerable due to their highly distributed and open nature.
  • Incentives: Many people must be convinced to volunteer their resources to applications, which can be a challenge.
