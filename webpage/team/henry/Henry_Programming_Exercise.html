Distance Vector Routing Algorithms

Distance Vector algorithms do exactly what they say on the box. They work in weighted graphs of nodes and edges to find the most efficient path between all nodes on the network. They are exhaustive, and have three key features:

DV algorithms are:

  1. Iterative: The algorithm runs until all neighbors have exchanged all pertinent information
  2. Asynchronous: Not all nodes will perform operations in lockstep with one another.
  3. Distributed: Each node gets information from its neighbor or neighbors.

In a DV algorithm, each node maintain a set of information to update and send to its neighbors. The information is:

  1. For each neighbor vertex (v), the cost (c(x, v)) to get from itself (x) to the each directly attached neighbor v.
  2. Dx: the estimated cost to get to all destinations (y), in N. This is the node's distance vector.
  3. The distance vectors of all its direct neighbors.

The Bellman Ford Least Cost Path runs as follows:

dx (y) = minv{c(x, v) + dv (y)}

  • dx (y) is the value of the least cost path from the current node to the target node (x to y).
  • minv represents a minimum distance between the current node and one of its direct neighbors.
  • Based on these components, the least cost path to get from the current node x to the destination node y will be represented by c(x, v) + dv (y).
